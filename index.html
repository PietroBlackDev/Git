Primeira pergunta

O que é Git?

Criado por Linus Torvalds (o mesmo criador do Linux) em 2005, o Git é o sistema de controle de versão mais usado no mundo, sendo rápido e efetivo para variados tipos de projetos na vida profissional, além de ser completamente distribuído. É um Sistema de Controle de Versão Distribuído, ou seja, ao invés de um único local com o histórico completo de uma versão, cada cópia do trabalho também possui um repositório que contém um histórico de todas as alterações. Diferente de outros VCS, o Git trata como um fluxo do estado, por meio do comando commit, ele “tira uma foto” do repositório e cria um link para ele.

As operações do Git são realizadas, em sua maioria, em sistemas locais, tornando muito melhor trabalhar com os arquivos mesmo na falta de conexão com a rede. Ele possui integridade até as suas bases, passando por verificações, em resumo, o usuário fica bem informado sobre o processo. O Git possui três estados fundamentais: committed, em que os dados estão armazenados de forma segura no banco de dados; modified, com o arquivo modificado, mas sem o comando commit ter sido dado; e staged, para marcar um arquivo para o próximo commit.

Segunda e terceira pergunta

O que é controle de versão?

Controle de versão é uma prática que se resume na alteração e gerenciamento do código, mantendo versões anteriores de um projeto ou qualquer tipo de arquivo. Logo, caso ocorra um erro e o usuário perca o arquivo atualizado, ele pode recuperá-lo em uma versão anterior. Da mesma forma, pode comparar as mudanças ao longo do tempo e verificar quem está mexendo.

O Controle de Versão pode ter sistemas locais, que armazenam as alterações no arquivo em um banco de dados simples, com controle de revisão; sistemas centralizados, em que um servidor central monitora alteração de usuários em computadores diferentes - embora, se sofrer um problema, ele afeta o andamento em todas as máquinas; ou sistemas distribuídos (Git), em que os clientes podem duplicar o repositório completo e até restaurá-lo caso o original seja apagado. É como um backup.

O que são branches?

Branches são nada mais do que ramificações existentes em qualquer sistema de Controle de Versão, divergindo do código principal sem alterá-lo. Embora seja um processo trabalhoso para outros sistemas, o Git é desenvolvido para os fluxos de trabalho mais ramificados e fluídos, tornando a criação de branches em algo muito mais leve. O branche no Git é basicamente criar um ponteiro para um determinado commit que foi criado, podendo alternar entre diferentes branches.

Qual a importância do Git para um grupo de faculdade?

Um grupo de faculdade pode usar o Git para garantir que nenhuma parte de um projeto seja perdido, pois, com o Git, ao invés de um único local com o histórico completo de uma versão, cada cópia do trabalho também possui um repositório que contém um histórico de todas as alterações. Dessa forma, é possível recuperar versões anteriores. Além de também servir para otimizar tempo, já que não seria mais preciso que as pessoas se encontrassem para mexer no código em um mesmo computador, porque com o Git é possível codificar de locais diferentes um mesmo projeto em tempo real.

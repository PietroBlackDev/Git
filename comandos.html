h1 - primeira pergunta

Como Configurar o Git?

A configuração do usuário é feita pelo git config, que vê e atribui variáveis de controle e configuração para todos os aspectos do Git, valendo uma vez por todas. Ou seja, quando você determina as configurações, elas vão continuar as mesmas após atualizações, sem necessidade de reescrevê-las.

O git config pode ser acessado por:

  • /etc/gitconfig, valendo para todos os usuários e repositórios;
  • ~/.gitconfig para o seu usuário;
  • pelo comando .git/config em um repositório qualquer.

Por ele, você pode configurar seu nome de usuário e o seu email. Também pode escolher um editor de texto para abrir uma mensagem no Git, mas no Windows, essa opção já vem durante a instalação.

h2 - elementos do meio

h2 - Gitpull

O que é Gitpull?

Git pull é um comando útil para quem trabalha no desenvolvimento de projetos em equipe e precisa estar sempre com a última versão do arquivo. É um atalho que permite verificar os repositórios remotos (git fetch) e gerar um arquivo final atualizado localmente (git merge).

Git pull é um comando no qual tem a função de buscar e baixar conteúdos de repositórios remotos e fazer a atualização ao repositório local para que os conteúdos de ambas fiquem iguais.

Como dito, o git pull é uma combinação de dois comandos, git fetch seguido por git merge.

No primeiro estágio, é executado o git fetch, que baixa o conteúdo do repositório remoto necessário.

Em seguida, o comando git merge combina várias sequências de commits em uma única ramificação.

h2 - mensagem errada commit

Como arrumar uma mensagem errada em um Commit?

No caso de mensagem do commit mais recente que tenha erros ou foi escrita imprecisamente, é possível alterar através do comando “git commit --amend”.

Se o commit ainda não foi enviado ao github através do push online, também é possível alterar a mensagem pelo comando “git commit --amend”.

  1. Navegue até o repositório em que o commit que você quer alterar está por meio da linha de comando;
  2. Digite “git commit --amend” e aperte Enter;
  3. No editor de texto corrija a mensagem e salve.

h2 - comandos do Git

Principais Comandos do Git

Git help – Pede ajuda sobre determinados comandos.

Git config – Configura suas informações de usuário e preferências

Git init – Cria repositórios.

Git status – Verifica estado de repositórios e arquivos.

Git clone – Clonar um repositório já existente por meio de um link.

Git add – Adiciona diretórios e/ou arquivos.

Git commit – Armazena o conteúdo atual num commit e mostra uma mensagem com as mudanças.

Git rm – Remove diretórios e/ou arquivos

Git mv – Renomeia e move arquivos.

Git branch – Cria um novo branch.

Git checkout – Ajuda a mudar de branch.

Git remote – Exibe informações de um repositório remoto.

Git push – Transfere as informações do repositório local para um remoto.

Git pull – Incorpora as atualizações de um repositório remoto no branch em que o usuário está.

Git log – Mostra o histórico de commits criados.
